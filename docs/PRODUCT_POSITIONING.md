# Product Positioning

How Nova Systems Lab is positioned publicly, and the rules that keep that
positioning truthful. Voice and wording live in `BRAND_IDENTITY.md`.

## Positioning statement

Nova Systems Lab is an independent, founder-led, engineering-first software
organization. It builds systems tools, developer utilities, platform
integrations, and experimental runtime technologies, developed in the open, with
the intent to become a commercial software company as the work matures.

## Present as

- a growing modern technology startup
- an engineering-led product organization
- an independent open-source software organization
- a **future** commercial software company
- a builder of systems tools, developer utilities, runtime technologies,
  platform integrations, and experimental software

## Never present as

- a personal portfolio
- a student-project collection
- a generic software agency
- a donation-driven nonprofit
- an NGO
- a Section 8 company
- a registered company (unless registration actually occurs)
- a funded startup
- a multinational organization
- a large established corporation

Nova Systems Lab is not currently a registered or incorporated entity, and no
copy may imply otherwise. It does not solicit donations.

## Organizational strategy

Build demonstrable products first and let them establish credibility. Begin with
selected small clients. Do not solicit donations. Support open-source
development where appropriate. Preserve the ability to commercialize later.
Register formally when appropriate. Consider external investment only after
product and organizational maturity.

The public site should reflect this order: **products and progress first**,
commercial framing later.

## Product status vocabulary

Every publicly listed project carries exactly one status. This vocabulary is
implemented as the `ProductStatus` type in the `StatusBadge` primitive.

| Status            | Means                                             |
| ----------------- | ------------------------------------------------- |
| Research          | Exploring a problem space; no committed direction |
| Concept           | A defined idea; not yet planned                   |
| Planning          | Scope and approach being defined                  |
| Foundation        | Groundwork being built; nothing usable yet        |
| Early development | Actively being built; not usable                  |
| Prototype         | Runs; throwaway or proof-of-concept quality       |
| Private preview   | Usable by invited testers only                    |
| Public preview    | Publicly testable; expect breakage                |
| Beta              | Feature-complete; stabilising                     |
| Stable            | Production-ready and supported                    |
| Maintenance       | Supported; no active feature work                 |
| Paused            | Work intentionally stopped; may resume            |
| Archived          | No longer developed or supported                  |

**Rule:** never label a project Stable, Beta, or Preview without evidence
(a real release, a real tester group, a real support commitment). When in doubt,
choose the _less_ mature label. A status must be assigned by the founder, not
inferred.

At the time of writing, **no product status has been assigned** to WSL Studio or
WinDroid Runtime. Until one is, public copy may say only that they are in
development — the wording approved in `BRAND_IDENTITY.md`.

## Showing a growing organization

Growth is demonstrated with truthful, verifiable signals only:

- mission completions
- changelog entries
- releases
- active milestones
- documentation updates
- public roadmap items
- contribution activity
- current project status

Never use fake scale indicators: invented user counts, download totals, customer
logos, testimonials, funding badges, or team photos. If a signal cannot be
sourced from the repository or a real release, it does not go on the site.

product support, integrations, custom internal tooling, and platform-development
services — without becoming a generic agency website. Products stay primary.
